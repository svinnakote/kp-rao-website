# AGENTS.md

## Operating Rules
Before making major changes:
1. Propose a plan first.
2. Prefer small diffs.
3. Ask before introducing frameworks.

## Do
- Default to plain HTML/CSS.
- Prefer modular sections.
- Keep code simple and readable.
- Use semantic markup.
- Make responsive by default.
- Prefer elegant whitespace over complexity.
- Suggest improvements when helpful.

## Don't
- Do not introduce React/Next/Tailwind unless asked.
- Do not over-engineer.
- Do not use generic startup landing page aesthetics.
- Do not add unnecessary animations.
- Do not use fintech cliches or crypto style visuals.

## Design Preferences
Favor:
- Trust over hype
- Professional over trendy
- Premium simplicity over feature-heavy layouts

## Working Style
When asked for changes:
- Explain plan briefly
- Then implement

For larger tasks:
- Break into steps
- Complete step-by-step
